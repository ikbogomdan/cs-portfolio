# linked-list


## Описание

Как вы знаете, в регулярных файлах не всегда хранятся текстовые данные.
Например есть картинки, музыка, видео, программы...

А для обмена и хранения данных (продукты в магазине, список фильмов
в кинотеатрах, ...) не всегда используется текстовые json, csv, xml и им подобные.
Существуют бинарные форматы, такие как BSON, Apache Avro и Procol Buffers и другие.

Сейчас потренируемся работать с данными в бинарном представлении


## Задание

На вход функции `read_linked_list` подается путь до файла, в котором в
бинарном виде хранится связанный список.

Связанный список хранится блоками (1 блок - 1 нода) вида
- 4 байта занимает смещение от начала файла следующей ноды; -1 - конец списка
- 2 байта - двухбуквенный код страны (ru / kz / ge / uk / am / ...)
- 4 байта - артикул товара

При записи в файл, каждая нода записывалась **компактно** (без паддингов).

Гарантируется, что первая нода находится по нулевому смещению

Вам нужно правильно прочитать связанный список из файла и собрать список из
структур `ListNode`, затем вернуть из функции его начало.
Весь код в `linked_list.c` можно изменять.

Не забывайте обрабатывать ошибки.
В случае ошибки достаточно завершить программу с кодом возврата != 0.

В своём решении используйте только `read` и `lseek` для чтения из файла.
Использовать `FILE*` запрещено :)
