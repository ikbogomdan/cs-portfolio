# Условие задачи: StackAllocator и Allocator-Aware List

Задача состоит из двух частей: написание собственного аллокатора, работающего на стековой памяти, и реализация двусвязного списка, умеющего правильно работать с произвольными аллокаторами.

---

## Часть 1. StackAllocator

Напишите класс `StackAllocator<typename T, size_t N>`, который бы позволял создавать стандартные контейнеры на стеке, без единого обращения к динамической памяти. 

### 1. StackStorage
Для этого вам пригодится написать класс `StackStorage<size_t N>`, в котором будет храниться большой массив на стеке.
* Объект `StackStorage` **не должен** поддерживать копирование.
* Во время исполнения может существовать несколько `StackStorage` с одним и тем же `N`. Аллокаторы, построенные на разных `StackStorage`, должны считаться неравными.

### 2. StackAllocator
Объект класса `StackAllocator` должен быть легковесным, легко копироваться и присваиваться. Он должен удовлетворять требованиям к аллокатору (STL-совместимым) и позволять использование в качестве аллокатора для стандартных контейнеров.
Должны быть определены:
* Конструктор копирования, деструктор, оператор присваивания.
* Методы `allocate`, `deallocate`.
* Внутренний тип `value_type`.
* Метод `select_on_container_copy_construction`, если логика работы вашего аллокатора этого потребует.

**Требования к памяти и выравниванию:**
* `StackAllocator` должен заботиться о выравнивании объектов (`alignof`). В частности, нельзя класть переменные типа `int` по адресам, не кратным 4; переменные типа `double` — по адресам, не кратным 8, и т.д.
* Использование любого контейнера со `StackAllocator` вместо `std::allocator` должно приводить к тому, что обращений к динамической памяти в программе не происходит вообще.

**Пример использования:**
```cpp
int main() {
    StackStorage<100'000> storage;
    StackAllocator<int, 100'000> alloc(storage);
    std::vector<int, StackAllocator<int, 100'000>> v(alloc);
    // ... useful stuff ...
}
