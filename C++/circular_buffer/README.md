# Условие задачи: CircularBuffer

Реализуйте кольцевой буфер `CircularBuffer`, который должен иметь 2 шаблонных аргумента: тип хранимых значений `typename T` и `std::size_t Capacity = DYNAMIC_CAPACITY` (где `DYNAMIC_CAPACITY` равна `std::numeric_limits<std::size_t>::max()`).

Пользователь, если ему известна вместимость буфера на этапе компиляции (compile-time), может указать её как шаблонный аргумент `Capacity`, иначе она будет равна `DYNAMIC_CAPACITY`, что означает, что вместимость известна лишь во время выполнения (runtime).

## Два режима работы
Возникают 2 разных случая, в которых ваш `CircularBuffer` должен вести себя по-разному:

### 1. Если Capacity == DYNAMIC_CAPACITY:
* Буфер должен аллоцироваться в динамической памяти.
* Известная лишь в рантайме вместимость хранится полем.

### 2. Если Capacity != DYNAMIC_CAPACITY:
* Буфер должен создаваться на стеке, вы не должны делать никаких запросов на аллокацию динамической памяти.
* Запрещается хранить вместимость полем.

**Важно:** Вы не должны дважды полностью реализовывать `CircularBuffer` под случаи, когда вместимость известна в компайл-тайме и когда нет. Избежать огромного количества копипасты — часть вашей задачи.

*Разрешается использовать `std::array`. Другие стандартные контейнеры использовать нельзя.*

---

## Итераторы
Должен быть внутренний тип `iterator` (с маленькой буквы, в традициях STL), поддерживающий:
1. Инкремент (++), декремент (--)
2. Сложение с целыми числами
3. Сравнение (<, >, <=, >=, ==, !=)
4. Взятие разности двух итераторов
5. Разыменование (унарная звёздочка *), результатом является `T&`
6. Оператор ->, результатом является `T*`

**Дополнительные требования к итераторам:**
* Должен быть константный итератор `const_iterator`, не позволяющий менять элемент. Разрешена конверсия (в том числе неявная) из `iterator` в `const_iterator`, обратно — нет. Реализовать его нужно так, чтобы код не являлся копипастой обычного `iterator`.
* Должны быть реверс-итераторы: `reverse_iterator` и `const_reverse_iterator`.
* Если `Capacity != DYNAMIC_CAPACITY`, хранить в итераторе вместимость буфера полем **нельзя**.
* Хранить в итераторе указатель на сам объект `CircularBuffer` **нельзя**: разыменование итератора должно требовать лишь одного разыменования указателя!
* *Подсказка:* При реализации итератора вам может быть полезен стандартный класс `std::span`.

## Методы контейнера
*При реализации методов можете считать, что вместимость буфера всегда положительна (отдельно обрабатывать случай нулевой вместимости не обязательно).*

### Конструкторы
**Если Capacity == DYNAMIC_CAPACITY:**
* Должен быть конструктор, принимающий `std::size_t capacity`. Не должно быть конструктора по умолчанию.

    CircularBuffer<int> buffer1{32}; // OK
    CircularBuffer<int> buffer2{};   // Не OK, ошибка компиляции (вместимость не задана)

* Должен быть конструктор копирования от другого буфера такого же типа и оператор присваивания.

**Если Capacity != DYNAMIC_CAPACITY:**
* Должен быть конструктор по умолчанию и конструктор, принимающий `std::size_t capacity`. Во втором случае аргумент `capacity` должен быть равен `Capacity`, иначе нужно кидать исключение `std::invalid_argument`.

    CircularBuffer<int, 32> buffer1{};   // OK
    CircularBuffer<int, 32> buffer2{32}; // OK
    CircularBuffer<int, 32> buffer3{42}; // Не OK, выбросит std::invalid_argument

* Должны быть конструктор копирования и оператор присваивания от другого буфера такого же типа (такой же вместимости).

    CircularBuffer<int, 32> buffer1{};
    CircularBuffer<int, 32> buffer2{buffer1}; // OK
    CircularBuffer<int, 42> buffer3{buffer1}; // Не OK, ошибка компиляции
    buffer1 = buffer2; // OK
    buffer1 = buffer3; // Не OK, ошибка компиляции

### Основные методы
* **Вставка и удаление (O(1)):**
  * `push_back`, `push_front`. В случае, если буфер уже полон, должно происходить затирание краевого значения (для `push_back` затирается первый/передний элемент, для `push_front` — последний/задний).
  * `pop_back`, `pop_front`.
* **Информация о состоянии:** `size()`, `capacity()`, `empty()`, `full()` (возвращает `true` если и только если буфер полностью заполнен).
* **Доступ к элементам (O(1)):**
  * Оператор `[]` (без проверок выхода за границу).
  * `at` (кидает `std::out_of_range`).
* **Методы итераторов:**
  * `begin`, `cbegin`, `end`, `cend`. Если сам контейнер константный, `begin` и `end` возвращают константные итераторы. Декремент `end` должен давать итератор на последний элемент, вычитание целых чисел из `end` также должно корректно работать.
  * `rbegin`, `rend`, `crbegin`, `crend`.
* **Модификация:**
  * `swap(other)`: С другим буфером того же типа. **Важно:** если `Capacity == DYNAMIC_CAPACITY`, то существовавшие до этого итераторы должны продолжать указывать на те же элементы.
  * `insert(iterator, const T&)` (линейное время): Делает вставку значения перед элементом, на который указывает итератор. Элемент и все остальные правее сдвигаются на один вправо. При отсутствии свободного места затирается первый элемент в контейнере. Если места нет и итератор указывает на первый элемент, вставка не происходит, содержимое остаётся как было.
  * `erase(iterator)` (линейное время): Удаляет элемент по итератору. Все элементы справа сдвигаются на один влево.

---

## Исключения (Exception Safety)
Все методы, кроме `insert` и `erase`*, должны быть **строго безопасны** относительно исключений. Это значит, что в случае исключения в конструкторе или операторе присваивания типа `T` во время выполнения какого-либо метода `CircularBuffer`, контейнер должен вернуться в исходное состояние (которое было до начала метода) и пробросить исключение наверх в вызывающий код.

*\* В случае, когда `Capacity != DYNAMIC_CAPACITY`, к методам, которые не должны быть строго безопасны относительно исключений, прибавляются оператор `=` и `swap`.*
